pk-textarea

A rich text editor built entirely with native contenteditable + execCommand — no external packages. Supports bold/italic/underline/strikethrough, text colour, font name, font size (including headings), ordered/unordered lists, dark theme, and three view modes (Edit / HTML / Plain text). Implements ControlValueAccessor for use with [(ngModel)] or reactive forms.

API — inputs

Input Type Default Description
placeholder string 'Type something...' Placeholder text shown when the editor is empty
theme 'light'|'dark' 'light' Color theme
showToolbar boolean true Show or hide the formatting toolbar on init
minHeight string '200px' Minimum height of the editable area (any CSS value)
disabled boolean false Disable editing (also settable via reactive-form disable())
customClass string '' Extra CSS class on the root element
customStyle Record<string,string>|null null Inline style object on the root element
[(ngModel)] PkTextareaValue — Two-way binding via ControlValueAccessor — value has .html and .text

API — toolbar features

Feature Type Default Description
Bold / Italic / Underline / Strike — — Active state tracked via queryCommandState
Text colour — #000000 Native <input type="color"> — selection is preserved across the dialog
Highlight color — #ffff00 Text background color — uses execCommand('hiliteColor'); selection preserved same way as text color
Blockquote — — Toggle <blockquote> block via execCommand('formatBlock'); active state tracked; click again to revert to <p>
Font name — Default 19 options — applies pk-font-* class to selected text via execCommand trick
Font size PkTextareaFontSize — small / normal / large / h1 / h2 / h3
Ordered / Unordered list — — Uses execCommand('insertOrderedList') / insertUnorderedList
View mode 'edit'|'html'|'text' 'edit' Edit / HTML source / Plain text — toggled at top-right of toolbar
Toolbar toggle — — Hamburger button always visible — hides/shows the entire toolbar row

Notes

  • Zero external dependencies — pure Angular + native browser APIs.
  • execCommand is deprecated but still works in all major browsers; it remains the only standard way to implement rich text in contenteditable.
  • The toolbar uses mousedown + preventDefault to keep the text selection alive while clicking format buttons.
  • For the colour picker (native dialog), the last known selection is saved via the document:selectionchange listener and restored before applying foreColor.
  • Content is preserved in the DOM across mode switches (HTML / text views are read-only mirrors).
