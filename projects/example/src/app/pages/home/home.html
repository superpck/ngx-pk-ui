ngx-pk-ui

An Angular 21 component library built for modern applications. It provides ready-to-use UI components and CSS utilities designed around Angular's signal-based architecture — fully standalone, no NgModules required.

GitHub Live Demo npm

Usage

1. Install

npm install ngx-pk-ui

2. Add global styles (angular.json)

"styles": ["node_modules/ngx-pk-ui/styles/pk-ui.css"]

3. Import components

import { PkToastr, PkToastrService, PkAlert, PkAlertService, PkTooltip } from 'ngx-pk-ui';

@Component({
  imports: [PkToastr, PkAlert, PkTooltip],
  // ...
})
export class AppComponent { }

4. Inject and use

import { PkToastrService, PkAlertService } from 'ngx-pk-ui';

@Component({ ... })
export class MyComponent {
  toastr = inject(PkToastrService); // toast container auto-mounts
  alert  = inject(PkAlertService);  // dialog overlay auto-mounts
}

Notes

  • Angular 21+ required. This library uses signal(), input(), contentChildren(), and control-flow blocks (@for, @if). It is not compatible with Angular 16 or earlier.
  • Standalone only. There are no NgModules. Every component and directive must be added to the consuming component's imports[] array directly.
  • pk-toastr and pk-alert self-mount automatically when their service is first injected — no template tag required. If you still have <pk-toastr /> or <pk-alert /> in your template, the service detects it and skips the auto-mount to avoid duplication.
  • CSS is separate from components. Buttons, badges, cards, spinners, and grid classes are pure CSS and require pk-ui.css (or individual files) to be loaded globally. They do not auto-load by importing an Angular component.
  • Peer dependency: @angular/core ^21.0.0. When updating Angular, ensure the library version matches.
